# Финальная проверка структуры базы данных

## Результат

Структура базы данных первой версии признана финальной и достаточной для реализации описанных сценариев без добавления недокументированных таблиц или полей.

## Проверенное покрытие

| Требование | Представление в схеме |
|---|---|
| Одна запись без дубликатов между разделами | одна строка `records`, состояние и временной список в её полях |
| Исходный текст или голос Telegram | неизменяемая строка `source_messages`, связанная с `records` один к одному |
| «Входящие» и «Разобранные» | значения состояния `records`, без отдельных таблиц |
| «Сегодня», «Завтра» и «Неделя» | поле временного списка задачи в `records` |
| Выполнение, скрытие и восстановление | временные поля текущего и предкорзинного состояния `records` |
| Корзина максимум из 30 записей | `records.trashed_at`, стабильная сортировка и каскадное физическое удаление превышения |
| Несколько тегов у нетасковой записи | `tags` и связь многие ко многим `record_tags` |
| Разобранная запись без тегов после удаления последнего тега | состояние `records` сохраняется независимо от наличия строк `record_tags` |
| Результаты распознавания речи и получения метаданных ссылок без перезаписи | отдельные попытки `processing_results` |
| Повторная доставка Telegram | уникальная пара чата и сообщения в `source_messages` |
| Ежедневные операции без повторного применения | уникальный период в `scheduled_runs` |
| Не более одного удаляемого вечернего напоминания | `scheduled_runs.telegram_message_id` у актуальной вечерней проверки |
| Один вопрос о задаче старше восьми дней за период активности и удаление неотвеченного сообщения | `records.task_active_since`, `records.stale_prompted_at` и `records.stale_prompt_message_id` |
| Пошаговые диалоги после перезапуска | `bot_sessions` |
| Общий поиск и счётчики | документированные индексы; производные значения не дублируются в БД |

## Проверенные свойства целостности

- Все таблицы, поля, типы, обязательность, внешние ключи, уникальности, индексы и правила удаления описаны.
- Задача находится ровно в одном временном списке и не имеет тегов.
- Выполнение, полуночное скрытие и корзина не стирают временной список задачи; он нужен для отображения, восстановления и возобновления.
- Нетасковая разобранная запись не имеет временного списка; отсутствие тегов допустимо только как результат последующего удаления всех её тегов.
- Разбор входящей записи меняет существующую строку и не создаёт копию.
- Корзина сохраняет прежнее состояние для восстановления, пока запись входит в последние 30.
- Окончательное удаление самой старой записи очищает зависимые строки и не оставляет висячих внешних ключей.
- Исходные пользовательские данные и результаты автоматической обработки разделены.
- Секреты, HTTP-заголовки и производные счётчики в базе не хранятся.

## Отложенные решения без изменения структуры

Статусы, теги, два типа записей и хранение голоса уже определены без расширения схемы. Метаданные ссылок используют существующие `source_messages`, `processing_results` и текущий текст `records`, поэтому новая таблица или новое поле не требуется.
