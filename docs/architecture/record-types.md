# Типы записей

В первой версии существуют ровно два типа записи: `thought` («Мысль») и `task` («Задача»). Покупка, ссылка, книга, фильм, игра, идея и прочее не являются отдельными типами; это содержание мысли, которое различается тегами.

## `thought` — «Мысль»

«Мысль» — любая запись, которая в текущем состоянии не является задачей временного списка.

К мыслям относятся личные наблюдения, размышления, выводы, заметки, идеи без срока, покупки без срока, ссылки, книги, фильмы, игры и другие сохранённые материалы.

Правила:

- сообщение без начального префикса «Сегодня», «Завтра» или «На этой неделе» создаёт `record_type = thought` в `inbox`;
- разбор по тегам сохраняет тип `thought` и переводит запись в «Разобранные»;
- мысль не имеет `task_list`, `completed_at`, `task_active_since` и `hidden_at`;
- мысль во «Входящих» не имеет тегов, а разобранная мысль может иметь несколько тегов или остаться без тегов после их удаления;
- действие «Сделать задачей» меняет существующую мысль на `task` и снимает все теги;
- обычную задачу обратно в мысль превратить нельзя; единственное исключение — исправление только что созданной префиксом задачи через кнопку подтверждения, пока это подтверждение активно.

## `task` — «Задача»

«Задача» — запись с действием, помещённая ровно в один временной список: «Сегодня», «Завтра» или «Неделя».

Правила:

- `record_type = task`, `lifecycle_state = task`, `task_list` содержит ровно одно из `today`, `tomorrow`, `week`;
- задача создаётся начальным временным префиксом либо подтверждённым превращением мысли;
- одна задача может содержать несколько пунктов в тексте, потому что одно Telegram-сообщение не делится автоматически;
- задача не имеет тегов и не показывается в «Разобранных»;
- выполнение, перенос, скрытие, корзина и возобновление меняют существующую строку и не создают копий.
- кнопка «Изменить» в подтверждении только что созданной префиксом задачи может один раз вернуть её в `thought`/`inbox`; после закрытия подтверждения обратного перехода нет.

## Ссылки внутри мыслей

Отдельно отправленная ссылка создаёт одну мысль во «Входящих». URL сохраняется неизменённым в исходном сообщении. При наличии интернета бот автоматически получает название страницы и краткое описание и добавляет их в отображаемое представление той же мысли; новая запись не создаётся. Формат обогащённого текста:

```text
<URL>
Название: <название>
Описание: <краткое описание>
```

Строка названия или описания пропускается, если соответствующего значения нет; исходный URL присутствует всегда.

Попытка получения метаданных сохраняется в `processing_results` с операцией `link_metadata`; URL остаётся в исходных данных, а полученные название и описание — в результате обработки. Отображаемый текст автоматически обогащается только если с момента сохранения ссылки владелец не изменял `records.display_text`. Если текст уже отредактирован, успешный результат сохраняется в `processing_results`, но пользовательская правка не перезаписывается.

Если сайт или интернет недоступны, мысль сохраняется сразу только с URL, а обработка повторяется после восстановления соединения. Ошибка метаданных не блокирует просмотр, разбор или изменение записи.

Автоматическая загрузка разрешена только для абсолютных публичных URL со схемой `http` или `https`. URL с учётными данными, `localhost`, локальными, частными, loopback, link-local, multicast и зарезервированными IP-адресами запрещены. DNS проверяется перед соединением и заново после каждого перенаправления; переход на запрещённый адрес не выполняется. Разрешено не более пяти перенаправлений, ожидание одного запроса ограничено 10 секундами, загружается не более 2 МБ и только HTML-ответ. Превышение ограничений и неподдерживаемый адрес считаются постоянной ошибкой и автоматически не повторяются.

Сетевой тайм-аут, временная ошибка DNS, разрыв соединения и HTTP `429` или `5xx` считаются временными ошибками. Они повторяются с увеличивающейся задержкой после восстановления работы, но одновременно существует только одна незавершённая логическая операция для записи. Остальные HTTP `4xx`, запрещённый адрес, неподдерживаемый тип содержимого и неверный URL считаются постоянными ошибками; исходный URL остаётся записью во «Входящих» без метаданных.

Если ссылка находится внутри сообщения с другим текстом, всё сообщение остаётся одной мыслью и автоматическое получение метаданных не запускается. В первой версии бот не разделяет ссылки и текст на отдельные записи и не извлекает несколько самостоятельных идей из одного сообщения.
