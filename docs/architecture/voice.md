# Голосовой ввод

## V1.0

В версии 1.0 бот не принимал голосовые сообщения и просил владельца использовать текст или
диктовку клавиатуры телефона.

## V1.1: Telegram voice message -> Deepgram -> текст

Во втором релизе бот принимает `voice` сообщения от единственного разрешенного пользователя и
расшифровывает их через Deepgram.

Поток обработки:

1. Бот получает Telegram `voice` message.
2. Проверяет идемпотентность по `chat_id + message_id` через общий механизм сохранения.
3. Скачивает голосовой файл из Telegram во временный файл.
4. Отправляет временный файл в Deepgram.
5. Удаляет временный файл сразу после распознавания.
6. Полученный текст передает в тот же сценарий, что обычное текстовое сообщение.

Правила сохранения результата совпадают с текстовым вводом:

- текст начинается с `Сегодня` — создать задачу в списке «Сегодня»;
- текст начинается с `Завтра` — создать задачу в списке «Завтра»;
- текст начинается с `На этой неделе` — создать задачу в списке «Неделя»;
- без префикса — сохранить мысль во «Входящие»;
- одно голосовое сообщение создает одну задачу или одну мысль;
- сохраненный исходный текст — это неизмененный текст, полученный от Deepgram;
- исходное аудио не хранится в базе данных и не остается в проектной папке.

Конфигурация:

- `DEEPGRAM_API_KEY` — обязательна для распознавания voice-сообщений;
- если ключ не задан или Deepgram недоступен, бот без звука сообщает, что голос не удалось
  распознать, и просит отправить текстом.

Ответы Telegram:

- все ответы отправляются с `disable_notification=True`;
- при успешном распознавании бот отвечает тем же подтверждением, что текстовый ввод: итоговый
  текст, место сохранения, кнопка «Изменить»;
- при пустой расшифровке бот без звука просит повторить текстом.

Deepgram выбран как внешний бесплатный стартовый сервис для `v1.1`. Архитектура не вводит
долгосрочную зависимость на хранение аудио: если бесплатный доступ закончится, можно заменить
только слой распознавания, не меняя правила сохранения записей.
