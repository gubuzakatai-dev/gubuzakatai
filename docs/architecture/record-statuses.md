# Статусы записей

Статусы описывают допустимые состояния одной строки `records`. Временной список задачи хранится отдельно в `task_list`, а выполнение — в `completed_at`; это предотвращает дублирование одной задачи между разделами.

## `inbox`

`inbox` — состояние неразобранной записи, которая требует решения владельца.

Запись получает `lifecycle_state = inbox`, когда принято текстовое или распознанное голосовое сообщение без явного префикса «Сегодня», «Завтра» или «На этой неделе».

Инварианты:

- запись ещё не является задачей: `task_list = NULL`, `completed_at = NULL`, `task_active_since = NULL`, `hidden_at = NULL`;
- у записи нет строк в `record_tags`;
- запись не находится в корзине: `trashed_at = NULL`;
- она учитывается в счётчике «Входящие» и показывается на соответствующем экране;
- время нахождения во «Входящих» не ограничено.

Допустимые переходы после подтверждения владельца:

- `inbox -> task` с одним из списков `today`, `tomorrow` или `week`;
- `inbox -> processed` с подтверждёнными тегами;
- `inbox -> trash` с сохранением `pre_trash_lifecycle_state = inbox`.

Переход выполняется над существующей строкой `records`, поэтому копия записи не появляется. После выхода из `inbox` вручную вернуть задачу или разобранную запись во «Входящие» нельзя. Единственное возвращение в `inbox` — восстановление записи, которая была отправлена в корзину непосредственно из «Входящих».
