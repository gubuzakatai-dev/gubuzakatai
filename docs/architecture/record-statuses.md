# Статусы записей

Статусы описывают допустимые состояния одной строки `records`. Временной список задачи хранится отдельно в `task_list`, а выполнение — в `completed_at`; это предотвращает дублирование одной задачи между разделами.

## `inbox`

`inbox` — состояние неразобранной записи, которая требует решения владельца.

Запись получает `lifecycle_state = inbox`, когда принято текстовое или распознанное голосовое сообщение без явного префикса «Сегодня», «Завтра» или «На этой неделе».

Инварианты:

- запись ещё не является задачей: `task_list = NULL`, `completed_at = NULL`, `task_active_since = NULL`, `hidden_at = NULL`;
- у записи нет строк в `record_tags`;
- запись не находится в корзине: `trashed_at = NULL`;
- она учитывается в счётчике «Входящие» и показывается на соответствующем экране;
- время нахождения во «Входящих» не ограничено.

Допустимые переходы после подтверждения владельца:

- `inbox -> task` с одним из списков `today`, `tomorrow` или `week`;
- `inbox -> processed` с подтверждёнными тегами;
- `inbox -> trash` с сохранением `pre_trash_lifecycle_state = inbox`.

Переход выполняется над существующей строкой `records`, поэтому копия записи не появляется. После выхода из `inbox` вручную вернуть задачу или разобранную запись во «Входящие» нельзя. Единственное возвращение в `inbox` — восстановление записи, которая была отправлена в корзину непосредственно из «Входящих».

## `today`

`today` — пользовательское состояние активной или отмеченной выполненной задачи списка «Сегодня». В базе это не отдельное значение жизненного цикла: запись имеет `lifecycle_state = task` и `task_list = today`.

Задача попадает в «Сегодня»:

- непосредственно из сообщения с явным префиксом «Сегодня»;
- при ручном или подтверждённом AI-разборе записи из `inbox`;
- при превращении разобранной нетасковой записи в задачу;
- при ручном переносе из `tomorrow` или `week`;
- автоматически из `tomorrow` в 00:00 по Москве.

Инварианты:

- `record_type = task`, `lifecycle_state = task`, `task_list = today`;
- у задачи нет строк в `record_tags`;
- активная задача имеет `completed_at = NULL`, `hidden_at = NULL` и заполненный `task_active_since`;
- перенос существующей задачи между списками не сбрасывает `task_active_since`;
- при создании задачи из `inbox` или «Разобранных» `task_active_since` получает время подтверждённого преобразования;
- задача учитывается в счётчике «Сегодня», пока не скрыта и не находится в корзине.

Превращение записи из «Разобранных» выполняется одной транзакцией: владелец выбирает «Сегодня», «Завтра» или «Неделя», все связи `record_tags` удаляются, существующая строка `records` становится задачей выбранного списка. Копия не создаётся.

Допустимые действия и переходы:

- отметить выполненной и отменить отметку до ближайшей полуночи;
- перенести в `tomorrow` или `week` без создания новой записи;
- отправить в корзину после подтверждения;
- изменить текст без изменения исходного сообщения.

В 00:00 по Москве активная невыполненная задача остаётся в `today`. Выполненная задача скрывается из активного списка. Если задача непрерывно не выполнена больше пяти дней, бот предлагает владельцу выбрать дальнейшее действие; само по себе достижение пяти дней состояние задачи не меняет.

## `week`

`week` — пользовательское состояние активной или отмеченной выполненной задачи списка «Неделя». В базе запись имеет `lifecycle_state = task` и `task_list = week`.

Список означает текущую календарную неделю по московскому времени: с понедельника 00:00 до воскресенья включительно. Для каждой задачи отдельная дата недели не хранится.

Задача попадает в `week`:

- непосредственно из сообщения с явным префиксом «На этой неделе»;
- при ручном или подтверждённом AI-разборе записи из `inbox`;
- при превращении разобранной нетасковой записи в задачу с выбором «Неделя»;
- при ручном переносе из `today` или `tomorrow`.

Инварианты совпадают с другими временными списками задач:

- `record_type = task`, `lifecycle_state = task`, `task_list = week`;
- у задачи нет строк в `record_tags`;
- активная задача имеет `completed_at = NULL`, `hidden_at = NULL` и заполненный `task_active_since`;
- перенос между списками не сбрасывает `task_active_since`;
- задача учитывается в счётчике «Неделя», пока не скрыта и не находится в корзине.

Допустимые действия и переходы:

- отметить выполненной и отменить отметку до ближайшей полуночи;
- перенести в `today` или `tomorrow`;
- отправить в корзину после подтверждения;
- изменить текст.

В 00:00 выполненная задача скрывается. Активная задача остаётся в `week`; в ночь с воскресенья на понедельник она становится задачей новой текущей недели без изменения строки и без сброса возраста.

Задачу из `week`, как и из любого другого временного списка, нельзя превратить обратно в разобранную запись с тегами или вернуть во `inbox`. Её можно только выполнить, перенести между временными списками, изменить или отправить в корзину.
