# Использование ChatGPT

## Какие задачи выполняет

В первой версии ChatGPT используется только как помощник при разборе одной выбранной мысли из «Входящих». Он не принимает решений вместо владельца и не изменяет пользовательский текст.

Для одной записи ChatGPT предлагает ровно один взаимоисключающий маршрут:

1. сделать запись задачей и выбрать один список — «Сегодня», «Завтра» или «Неделя»;
2. оставить запись мыслью, перевести в «Разобранные» и предложить один или несколько тегов.

При маршруте мысли модель сначала использует существующие теги. Если точного тега нет, она может предложить название нового пользовательского тега длиной до 15 символов; создание такого тега требует отдельного подтверждения владельца. «Прочее» предлагается только при отсутствии более точного варианта.

## Ограничения

- ChatGPT не переписывает, не исправляет, не сокращает и не дополняет текст записи.
- ChatGPT не делит одно сообщение на несколько записей или задач.
- ChatGPT не назначает теги задаче и не помещает тегированную мысль во временной список.
- ChatGPT не применяет предложение автоматически: владелец подтверждает его, изменяет вручную или отменяет.
- ChatGPT не удаляет запись и не отправляет её в корзину по собственной инициативе.
- Распознавание голоса, получение метаданных ссылки, переносы в 00:00, вечернее напоминание и проверка пяти дней выполняются отдельными сервисами без ChatGPT.

## Результат анализа

Каждый запрос создаёт `processing_results` с операцией `inbox_classification`. В `input_text` сохраняется точный текст, переданный модели; в `output_json` — валидный структурированный ответ:

- `route`: `task` или `thought`;
- `task_list`: `today`, `tomorrow` или `week` только для задачи;
- `tag_ids`: существующие предложенные теги только для мысли;
- `new_tag_names`: новые предложенные имена только для мысли.

Текстовый ответ модели не является пользовательской записью. Применение подтверждённого решения отмечается связью `record_tags.assigned_by = ai_confirmed` для тегов; изменение типа и списка выполняется над существующей строкой `records`.

При ошибке или недоступности ChatGPT запись остаётся во «Входящих». Владелец может повторить запрос или перейти к ручному разбору.

## Автоматический запуск

Автоматического запуска ChatGPT в первой версии нет.

В 21:00 по Москве планировщик без обращения к AI проверяет число мыслей во «Входящих». Если список не пуст, бот один раз за московскую календарную дату предлагает начать разбор и показывает варианты «Разобрать вручную» и «Разобрать с ChatGPT». Нажатие второго варианта открывает самую старую запись, но запрос к модели отправляется только после явного выбора «Разобрать с ChatGPT» для этой записи.

Если предложение проигнорировано, AI не запускается и повторного вечернего сообщения в тот же день нет. Если «Входящие» пусты, бот ничего не отправляет. Идемпотентность вечерней проверки обеспечивает `scheduled_runs`, а не результат AI.
